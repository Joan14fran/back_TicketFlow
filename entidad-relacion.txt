1. Users (Usuarios)
id (PK)
username (String)
email (String)
password (String)
role (String) - (Ej: 'agent', 'client') 
is_active (Boolean) 

2. Categories (Categorías)
id (PK) 
name (String) 

3. Tickets
id (PK) 
title (String) 
description (Text) 
status (String) - (Simplifiquemos: 'open', 'in_progress', 'closed') 
priority (String) - (La mantenemos: 'low', 'medium', 'high') 
category_id (FK → Categories) 
created_by (FK → Users) 
assigned_to (FK → Users, Nullable) 
created_at (DateTime) 
updated_at (DateTime) 

4. Comments (Comentarios)
id (PK) 
ticket_id (FK → Tickets) 
user_id (FK → Users) 
content (Text) 
created_at (DateTime)
